  name = payload.name || given_name + family_name || nickname

5. Set Password Invalid Token Fix
=================================

Current issue:
Forgot-password email arrives with reset link and token, but POST /auth/set-password returns "Invalid token".

Investigate and fix.

Likely causes:

* token in URL is URL-encoded/decoded incorrectly
* frontend sends full URL instead of token
* token hash mismatch
* forgot-password stores hashed token but set-password compares raw token incorrectly
* token expires incorrectly
* token field overwritten
* old token cleared too early

Requirements:

* Forgot-password generated token must be accepted by /auth/set-password.
* Registration setup token must still work.
* /auth/set-password should accept:
  { "token": "rawToken", "password": "..." }
* It should also tolerate token accidentally sent as full URL:
  "http://localhost:5173/set-password?token=rawToken"
  and extract token safely.
* Validate expiry.
* Hash token using same hash function used when storing.
* After success:

  * update passwordHash
  * emailVerified = true
  * clear passwordSetupToken
  * clear passwordSetupTokenExpiresAt
  * return token + user like login

Add logs in dev only:

* token hash lookup attempted
* token found / not found
  Do NOT log raw token in production.
