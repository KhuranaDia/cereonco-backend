Goal:
Add direct messaging between users.

Required features:

1. Conversations
- Create or get conversation between two users.
- List current user's conversations.
- Conversation should show the other participant, last message, unread count, and updatedAt.

2. Messages
- Send message in a conversation.
- Get messages in a conversation.
- Mark messages as read.
- Unread message count.

3. Database tables
Add tables if missing:

conversations:
- id
- user_one_id
- user_two_id
- created_at
- updated_at

messages:
- id
- conversation_id
- sender_id
- receiver_id
- content
- media_urls text array default []
- is_read boolean default false
- created_at
- updated_at

4. API endpoints

POST /messages/conversations
Body:
{
  "recipientId": 2
}

GET /messages/conversations

GET /messages/conversations/:conversationId

POST /messages/conversations/:conversationId
Body:
{
  "content": "Hello",
  "mediaUrls": []
}

PATCH /messages/conversations/:conversationId/read

GET /messages/unread-count

5. Rules
- User must be authenticated.
- Users cannot access conversations they are not part of.
- Users cannot message themselves.
- If conversation already exists between same two users, return existing one.
- Messages newest/oldest order should be consistent; use oldest first inside a conversation.
- Conversation list should be newest updated first.
- Mark read should only mark messages sent by the other user.
- mediaUrls optional and should default to [].
- Keep response format:
{
  "success": true,
  "message": "",
  "data": {}
}

Error:
{
  "success": false,
  "message": ""
}
