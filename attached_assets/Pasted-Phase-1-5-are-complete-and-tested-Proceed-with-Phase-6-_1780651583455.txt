Build a complete notification system integrated with existing modules.

Scope:

EntityType:

* post
* comment
* group
* group_post
* user

Requirements:

1. Create notifications automatically when:

   * User A likes User B's post
   * User A comments on User B's post
   * User A replies to User B's comment
   * User A joins a group
   * User A creates a group post

2. Never notify users about their own actions.

3. Return actor information with notifications:

   * id
   * name
   * role
   * profilePhotoUrl

4. Notifications should be sorted newest first.

5. Add unread count support.

APIs:

GET /api/notifications

GET /api/notifications/unread-count

PATCH /api/notifications/:id/read

PATCH /api/notifications/read-all

Rules:

* JWT authentication required.
* Users can only access their own notifications.
* Marking an already-read notification should be idempotent.
* read-all should update all unread notifications for the current user.

Response format:

Success:
{
"success": true,
"message": "...",
"data": {}
}

Error:
{
"success": false,
"message": "..."
}
