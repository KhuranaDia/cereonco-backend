## Task 3: Dynamic Frontend URL for Email Links

Current issue:

The backend uses a static FRONTEND_URL while generating:

* Password reset links
* Password setup links
* Any future email action links

Requirement from frontend:

If the request comes from:

http://localhost:5173

then generate links using:

http://localhost:5173

Otherwise use the TEST frontend URL (via environment variable).

---

### Preferred Logic

Determine frontend URL in this order:

1.

Use incoming request Origin header if present.

Example:

Origin:

http://localhost:5173

↓

Email link:

http://localhost:5173/set-password?token=...

---

2.

If Origin header is missing,

use

process.env.FRONTEND_URL

---

3.

If FRONTEND_URL is missing,

use

process.env.TEST_FRONTEND_URL

---

4.

If both are missing,

fallback to

http://localhost:5173

---

### Implementation

Find every place where email links are generated.

This includes:

* forgot password
* setup password
* invitation emails
* future auth emails if applicable

Create a reusable helper:

getFrontendBaseUrl(req)

which returns the correct frontend URL.

Do NOT duplicate logic.

---

### Security

Do NOT trust any value from request body.

Only use:

req.headers.origin

Ignore any body parameter attempting to override the frontend URL.
